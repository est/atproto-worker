## Context

需要一个 serverless 平台来运行 ATProto Worker。关键需求：

- WebSocket 支持（firehose subscribeRepos）
- 定时任务（cron，用于 interaction 同步）
- KV 缓存（journal 缓存）
- 免费额度适合个人使用
- 低延迟，全球分布

候选平台：

- Cloudflare Workers（Workers + Durable Objects + KV + Cron Triggers）
- Vercel Edge Functions（无 WebSocket 持久化、无 Cron、无 KV）
- AWS Lambda（冷启动、WebSocket 支持差、成本高）

## Decision

选择 Cloudflare Workers。

关键依赖：

- **Durable Objects**：WebSocket firehose 需要持久化连接管理
- **KV**：journal 缓存，避免每次请求都从 HTTP 拉取
- **Cron Triggers**：定时同步 interactions
- **Workers Hibernation API**：WebSocket 长连接不消耗 CPU 时间

## Consequences

优点：

- 免费额度充足（100k 请求/天）
- 全球边缘部署
- Durable Objects 原生支持 WebSocket
- 无需管理服务器

缺点：

- 与 Cloudflare 平台强绑定
- Durable Objects 有区域限制
- 不支持 Vercel 等其他平台
