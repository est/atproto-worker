## Context

项目有两套代码运行在不同环境：

- `src/` — Cloudflare Worker（V8 runtime）
- `cli/` — Node.js 本地 CLI

两套代码中存在大量重复实现：

- `generateTID()` 重复 3 次（src/utils.js、cli/seal.js、cli/journal.js）
- `cborEncode()` 重复 2 次（src/utils.js、cli/crypto.js），且 CLI 版本支持 `$link` 标签而 Worker 版本不支持
- `computeCID()` 重复 2 次
- `base32Encode()` / `base32Decode()` 重复 2 次
- `isValidDID()` vs `validateDid()` 两个同功能不同名函数

## Decision

创建 `src/shared.js` 作为共享工具模块，包含：

- TID 生成
- Base32 编解码（RFC 4648）
- CID v1 计算（dag-cbor + sha-256）
- CBOR 编解码（含 `$link` CBOR tag 42 支持）
- Varint 编码
- CAR 文件生成

`src/utils.js` 从 shared.js 重导出共享函数，保留 Worker 特有的解析/校验函数。

`cli/crypto.js` 从 shared.js 导入 CBOR 和 CID 实现，保留 Node.js 特有的密钥管理（`@noble/secp256k1`、base58）。

`cli/journal.js` 和 `cli/seal.js` 从 shared.js 导入 `generateTID`。

## Consequences

优点：

- 消除代码重复，单一实现
- CBOR `$link` 支持统一
- CID 计算保证 CLI 和 Worker 一致
- 新增工具函数只需实现一次

缺点：

- CLI 依赖 `src/` 目录（`import from '../src/shared.js'`）
- 需要确保 shared.js 不引入 Node.js 专有 API

## References

- `src/shared.js` — 共享工具模块
- `src/utils.js` — Worker 工具（重导出 + Worker 特有）
- `cli/crypto.js` — CLI 加密工具（重导出 + Node 特有）
