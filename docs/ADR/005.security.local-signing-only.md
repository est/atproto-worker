## Context

AT Protocol 要求每个 repo 的写操作都经过 secp256k1 私钥签名。传统 PDS 在服务端持有私钥并实时签名。

对于一个运行在 Cloudflare Workers 上的个人 PDS，需要决定私钥的存放位置。

## Decision

私钥**仅在本地 CLI 使用**，Worker 永不持有或使用私钥。

实现方式：

- `config.json`（包含私钥）仅存在于本地，已加入 `.gitignore`
- CLI (`cli/seal.js`) 使用 `@noble/secp256k1` 在本地签名
- 签名后的事件写入 `journal.ndjson`
- Worker 从 journal 读取已签名的事件，不再参与签名流程
- `wrangler.toml` 中只有 `OWNER_PUBLIC_KEY`（公钥），无私钥

## Consequences

优点：

- 私钥攻击面最小化（仅本地机器）
- Worker 被攻破不会泄露私钥
- 符合最小权限原则

缺点：

- 所有写操作必须在本地完成
- 不支持 Web UI 直接发帖（需要额外的签名方案）
- 密钥轮换需要更新 journal 中的签名

- `cli/seal.js` — 签名 CLI
- `cli/crypto.js` — 密钥管理和签名实现
- `config.json.example` — 配置模板（无私钥）
- `wrangler.toml` — 公钥配置
