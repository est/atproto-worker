## Context

AT Protocol 的 `subscribeRepos` 需要长连接 WebSocket。在 Cloudflare Workers 上，WebSocket 连接需要持久化状态管理。

两种 WebSocket 管理方式：

- A: 普通 Durable Objects（WebSocket 保持活跃会消耗 CPU 时间）
- B: Hibernatable WebSockets API（连接保持但不消耗 CPU 时间）

## Decision

使用 Durable Objects 的 Hibernation API。

实现要点：

- `wrangler.toml` 中添加 `compatibility_flags = ["websocket_hibernation"]`
- 使用 `state.acceptWebSocket(server)` 接受连接
- 实现 `webSocketMessage`、`webSocketClose`、`webSocketError` 回调
- 支持 backfill（客户端带 cursor 连接时补发历史事件）
- 支持 broadcast（新事件到达时推送给所有客户端）

## Consequences

优点：

- 空闲 WebSocket 不消耗 CPU 时间
- 支持大量并发连接
- 符合 Cloudflare 推荐实践

缺点：

- 需要 Hibernation API 兼容性标志
- 与 Cloudflare Durable Objects 强绑定
