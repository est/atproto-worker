## Context

当前部署流程需要 8 步：安装 Node.js → 克隆 → npm install → 配置 wrangler.toml → 生成密钥 → 设置 secrets → 部署 → 配置静态托管。

目标是将部署简化为 1-3 步，让非技术用户也能使用。

## Decision (Draft)

采用 Cloudflare Deploy Button 方案（方案 A），实现一键部署：

部署按钮自动完成：

1. 创建 Worker
2. 创建 R2 Bucket（存储 journal，依赖 ADR-012）
3. 创建 KV Namespace（缓存）
4. 创建 Durable Object（firehose）
5. 生成密钥对并存入 Secrets
6. 设置初始配置

用户后续流程：

1. 点击 "Deploy to Cloudflare"
2. 输入用户名
3. 开始发帖（通过 Web UI，依赖 ADR-013）

备选方案：GitHub Template + Actions（方案 B），通过 "Use this template" 创建仓库，GitHub Actions 自动部署。

## Consequences (Draft)

优点：

- 部署门槛极低
- 所有资源自动创建和绑定
- 无需手动配置

缺点：

- 强依赖 Cloudflare 生态
- 需要 R2 存储（ADR-012）和 Web UI（ADR-013）作为前置
- Deploy Button 机制可能有平台限制
