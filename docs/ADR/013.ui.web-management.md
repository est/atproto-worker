## Context

当前所有写操作（发帖、点赞、关注）只能通过命令行 CLI 完成。这要求用户：

1. 理解 atproto、DID、CID 等概念
2. 安装 Node.js 和依赖
3. 使用命令行操作

目标是让非技术用户也能使用。

## Decision (Draft)

实现基于 Web 的管理界面，嵌入 Worker 中提供服务：

- 首次访问引导：设置 DID、Handle
- 发帖界面：输入文本，点击发布
- 记录列表：查看已发布的 posts、likes、follows
- Journal 状态：事件数量、链完整性
- Firehose 状态：连接数、最近广播

技术方案：

- 纯 HTML/CSS/JS（无框架，保持简单）
- 复用现有 XRPC 端点 + 新增管理端点

新增端点：

- `POST /api/post` — 创建帖子
- `POST /api/like` — 点赞
- `POST /api/follow` — 关注
- `GET /api/journal` — journal 状态
- `POST /api/journal/validate` — 验证完整性

## Consequences (Draft)

优点：

- 降低使用门槛
- 可视化状态展示
- 无需命令行

缺点：

- Web UI 发帖需要解决签名问题（见 ADR-005）
- 增加 Worker 代码复杂度
- 需要额外的安全考虑（CSRF、认证等）
