## Context

需要构建一个 AT Protocol Personal Data Server (PDS)。传统 PDS 实现（如官方 `atproto` 包）需要数据库、服务端签名、持久化状态，运维复杂度高。

目标是构建一个**极简 PDS**，适合个人使用，可以免费运行在 Cloudflare Workers 上，无需数据库。

候选方案：

- A: 传统 PDS（D1 数据库 + 服务端签名）
- B: 事件溯源 + 静态文件 + 无状态 Worker

## Decision

选择方案 B：事件溯源静态发布架构。

核心设计：

- `journal.ndjson` 是唯一数据源（append-only event log）
- 所有写操作在本地 CLI 完成，Worker 保持只读
- Worker 状态由 journal 完全决定，无外部数据库
- 序列号（seq）从 journal 字节偏移量派生，确定性、单调递增

## Consequences

优点：

- 零数据库运维成本
- 私钥永不离开本地机器
- Worker 完全无状态，可随时重建
- 部署简单，适合个人免费托管

缺点：

- 不支持并发写入
- 不支持动态编辑（只能追加）
- Firehose blocks 为空（见 ADR-006）
- Interactions 持久化需要额外方案
