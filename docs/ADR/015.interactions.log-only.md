## Context

项目目标之一是接收其他用户对自己帖子的 interactions（likes、reposts、followers）。

当前实现（`src/interactions.js`）通过 cron 每 5 分钟从 Bluesky 公共 API 拉取 interactions，但只打印日志，没有持久化到任何存储。

## Decision

Interactions 当前仅抓取并记录日志，不持久化。

原因：

- 项目使用事件溯源架构（ADR-002），journal 是 append-only 的事件日志
- Interactions 来自外部，不是用户自己的写操作，不适合放入 journal
- 没有为 interactions 配置单独的存储（D1 或 KV）
- 持久化需要额外的读取接口和存储管理

当前行为：

- cron 每 5 分钟触发 `syncInteractions()`
- 遍历最近 20 个帖子，从 `public.api.bsky.app` 获取 likes 和 reposts
- 结果仅通过 `console.log` 输出

## Consequences

优点：

- 实现简单，不引入额外存储
- 日志可用于调试和监控

缺点：

- Interactions 数据不持久，重启后丢失
- 没有用户可见的 interactions 展示
- 每次 cron 都重新拉取，无增量更新
