## Context

AT Protocol 支持两种 DID 方法：

- `did:web`：基于域名，DID 文档由 Web 服务器托管
- `did:plc`：注册在 plc.directory，不由 Web 服务器控制

Worker 需要决定支持哪种身份模型。原始实现中 `/.well-known/did.json` 无条件生成 `did:web:${host}`，与 `OWNER_DID` 配置不一致。

## Decision

同时支持两种 DID 方法，通过 `OWNER_DID` 配置决定行为：

- `did:web:yourdomain.com`：
  - `/.well-known/atproto-did` 返回 `did:web:yourdomain.com`
  - `/.well-known/did.json` 返回完整的 DID 文档
- `did:plc:xxx`：
  - `/.well-known/atproto-did` 返回 `did:plc:xxx`
  - `/.well-known/did.json` 返回 404（文档由 plc.directory 托管）

两个端点返回一致的身份信息。

## Consequences

优点：

- 灵活支持两种 DID 方法
- `did:plc` 用户不会得到错误的 `did:web` 文档
- 身份信息一致性有保障

缺点：

- `did:web` 需要域名配置和 HTTPS
- `did:plc` 需要在 plc.directory 注册
